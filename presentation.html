Data Science Pipeline Architecture

1. Data Preprocessing & Feature Engineering

  • Text Preprocessing: Comprehensive NLP pipeline including lemmatization, stopwords removal, punctuation cleaning, and diacritic removal
  • Feature Engineering: TF-IDF vectorization with configurable n-gram ranges, min/max document frequency thresholds
  • Data Sampling: Stratified sampling across different label groups to handle class imbalance

2. Hybrid Modelling Approach

The system employs a two-stage hybrid architecture:

Stage 1: Unsupervised Clustering

  • K-means clustering to discover natural groupings in email content
  • Optimal cluster selection using silhouette score and elbow method
  • Cluster analysis with vocabulary extraction for interpretability

Stage 2: Multi-Label Classification

  • Individual Logistic Regression models for each target variable with class weight balancing
  • Multi-Label MLP (Neural Network) for simultaneous prediction across all targets
  • Feature combination: TF-IDF features + cluster assignments as additional features

Handling Data Quality Issues

  • Confidence-based null value imputation: Uses trained models to fill missing labels with confidence thresholds
  • Class imbalance mitigation: Balanced class weights in logistic regression

Advanced Features

1. Multi-Output Learning

  • MultiOutputClassifier wrapper for simultaneous prediction across all target variables
  • Label encoding for categorical targets with proper inverse transformation

2. Production-Ready Architecture

  • Batch prediction capabilities for processing multiple emails efficiently
  • Real-time inference through Streamlit interface
  • Model persistence using joblib for deployment

3. Interpretability & Analysis

  • Cluster vocabulary analysis to understand content themes
  • Feature importance through TF-IDF weights
  • Confidence scoring for prediction reliability

Performance Monitoring
